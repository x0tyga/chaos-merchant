# Research & Development (department definition)

Owns making the actual product/service better, and understanding the
technical/competitive landscape well enough to know what "better" means.
For AI-agent-driven businesses specifically, this includes the AI agents
themselves — building them, maintaining them, deciding when a new one is
worth building.

## Core responsibilities

- Product/service roadmap — what's being built next and why
- Competitive/technical landscape awareness (what competitors and the
  underlying technology are doing that matters)
- For AI-driven businesses: agent design, agent maintenance, and deciding
  build-vs-buy on new capabilities (see [[Agent-Registry]] for the running
  list across all businesses)
- Technical debt / known-issues tracking (this is where a "KNOWN_ISSUES.md"-
  style doc belongs, per business)

## Typical KPIs to track

- Feature/capability shipped per period
- Known-issues open vs. closed
- For AI-agent businesses: agent uptime/reliability, cost per agent run

## Cadence

Whatever fits the pace of the business — Chaos Merchant's own `HANDOFF.md`
pattern (rewritten at the end of each work session) is one working example
of a fast-moving R&D log; slower-moving businesses might do this monthly
instead.
